# Handoff — April 17, 2025: AWS Deployment Checklist

## Goal

Deploy the portfolio as a static site on **AWS CloudFront + S3**, with automated deploys from GitHub via GitHub Actions.

---

## Architecture

```
GitHub push (main) → GitHub Actions → npm run build → aws s3 sync dist/ → CloudFront cache invalidation
```

- **S3 bucket** — stores the built `dist/` output. Not publicly accessible; CloudFront reads via OAC.
- **CloudFront distribution** — CDN layer. Handles HTTPS, caching, edge delivery.
- **IAM OIDC provider** — lets GitHub Actions authenticate to AWS without storing long-lived keys.
- **IAM role + policy** — scoped permissions for the deploy workflow.

## AWS Setup Checklist

### 1. S3 Bucket

- [ ] Create an S3 bucket (e.g. `jeremybrachle-portfolio`)
- [ ] **Block all public access** — CloudFront will serve files, not S3 directly
- [ ] Enable versioning (optional but recommended for rollbacks)

### 2. CloudFront Distribution

- [ ] Create a CloudFront distribution with the S3 bucket as origin
- [ ] Use **Origin Access Control (OAC)** — not the legacy OAI
- [ ] Set default root object to `index.html`
- [ ] **Add custom error responses for SPA routing:**
  - 403 → `/index.html`, response code 200
  - 404 → `/index.html`, response code 200
- [ ] Update the S3 bucket policy to allow CloudFront OAC read access (the console generates this for you)
- [ ] Enable gzip/Brotli compression
- [ ] Set cache behavior: `CachingOptimized` managed policy is fine for static assets

### 3. HTTPS / Custom Domain (optional, do later)

- [ ] Request an ACM certificate in **us-east-1** (CloudFront requires certs in us-east-1)
- [ ] Add alternate domain name (CNAME) to the CloudFront distribution
- [ ] Point DNS (CNAME or Route 53 alias) to the `*.cloudfront.net` domain

### 4. IAM: GitHub Actions OIDC Auth

This lets GitHub Actions deploy without storing AWS access keys as secrets.

- [ ] Create an **IAM OIDC identity provider** for `token.actions.githubusercontent.com`
  - Audience: `sts.amazonaws.com`
- [ ] Create an **IAM role** (e.g. `github-actions-portfolio-deploy`) with trust policy:
  ```json
  {
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {
        "Federated": "arn:aws:iam::ACCOUNT_ID::oidc-provider/token.actions.githubusercontent.com"
      },
      "Action": "sts:AssumeRoleWithWebIdentity",
      "Condition": {
        "StringEquals": {
          "token.actions.githubusercontent.com:aud": "sts.amazonaws.com"
        },
        "StringLike": {
          "token.actions.githubusercontent.com:sub": "repo:jeremybrachle/portfolio:ref:refs/heads/main"
        }
      }
    }]
  }
  ```
- [ ] Attach an **inline policy** to the role:
  ```json
  {
    "Version": "2012-10-17",
    "Statement": [
      {
        "Effect": "Allow",
        "Action": ["s3:PutObject", "s3:DeleteObject", "s3:ListBucket", "s3:GetObject"],
        "Resource": [
          "arn:aws:s3:::jeremybrachle-portfolio",
          "arn:aws:s3:::jeremybrachle-portfolio/*"
        ]
      },
      {
        "Effect": "Allow",
        "Action": "cloudfront:CreateInvalidation",
        "Resource": "arn:aws:cloudfront::ACCOUNT_ID:distribution/DISTRIBUTION_ID"
      }
    ]
  }
  ```

### 5. GitHub Actions Workflow

- [ ] Create `.github/workflows/deploy.yml`:
  ```yaml
  name: Deploy to AWS
  on:
    push:
      branches: [main]
  permissions:
    id-token: write
    contents: read
  jobs:
    deploy:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4
        - uses: actions/setup-node@v4
          with:
            node-version: 20
        - run: npm ci
        - run: npm run build

        - uses: aws-actions/configure-aws-credentials@v4
          with:
            role-to-assume: arn:aws:iam::ACCOUNT_ID:role/github-actions-portfolio-deploy
            aws-region: us-east-1

        - run: aws s3 sync dist/ s3://jeremybrachle-portfolio --delete
        - run: |
            aws cloudfront create-invalidation \
              --distribution-id DISTRIBUTION_ID \
              --paths "/*"
  ```
- [ ] Replace `ACCOUNT_ID` and `DISTRIBUTION_ID` with real values after creating the AWS resources

### 6. Verify

- [ ] Push to `main`, confirm the Actions workflow passes
- [ ] Hit the `*.cloudfront.net` URL — all pages load
- [ ] Test SPA routing: navigate to a page, refresh — should not 404
- [ ] Confirm `public/vibe-machine/` assets are accessible

## Order of Operations

1. Fix the Vite build config + test locally
2. Create S3 bucket
3. Create CloudFront distribution with OAC
4. Set up IAM OIDC provider + deploy role
5. Add the GitHub Actions workflow
6. Push and verify
